# Paradise Phuket Frontend

React приложение для отображения глобального лидерборда Paradise 2026 | Phuket.

## Технологии

- **React 18** с TypeScript
- **Vite** для сборки
- **CSS Modules** для изолированных стилей
- **Google Fonts** (Bricolage Grotesque, Inter)

## Структура проекта

```
src/
├── components/          # Переиспользуемые компоненты
│   ├── Header/         # Компонент хэдера с модальными окнами
│   │   ├── Header.tsx
│   │   ├── Header.module.css
│   │   └── modals/     # Модальные окна хэдера
│   │       ├── ExploreUsModal.tsx
│   │       ├── QuestionsModal.tsx
│   │       └── UserMenuModal.tsx
│   └── Footer/         # Компонент футера с модальными окнами
│       ├── Footer.tsx
│       ├── Footer.module.css
│       └── modals/     # Модальные окна футера
│           ├── SupportModal.tsx
│           └── TermsModal.tsx
├── pages/              # Страницы приложения
│   └── Leaderboard/    # Главная страница лидерборда
│       ├── Leaderboard.tsx
│       ├── Leaderboard.module.css
│       └── components/ # Компоненты страницы
│           ├── HeroSection/
│           ├── TopStandings/

├── styles/             # Глобальные стили
│   └── global.css
└── assets/             # Изображения и SVG
```

## Архитектура

### Изолированный CSS
Каждый компонент имеет свой собственный CSS модуль (`.module.css`), что обеспечивает:
- Изоляцию стилей между компонентами
- Отсутствие конфликтов имен классов
- Легкость поддержки и редактирования

### Компонентная структура
- **Header** и **Footer** выделены в отдельные компоненты в отдельных папках
- Каждый компонент имеет свою папку с файлами `.tsx` и `.module.css`
- Модальные окна находятся в подпапках `modals/` внутри соответствующих компонентов

### Модальные окна
Все модальные окна созданы как пустышки (placeholder) и готовы к дальнейшей реализации:
- `ExploreUsModal` - модальное окно "Explore Us"
- `QuestionsModal` - модальное окно "Questions"
- `UserMenuModal` - модальное окно меню пользователя
- `SupportModal` - модальное окно поддержки
- `TermsModal` - модальное окно условий использования

## Установка и запуск

```bash
# Установка зависимостей
npm install

# Запуск dev сервера
npm run dev

# Сборка для продакшена
npm run build

# Превью продакшен сборки
npm run preview
```

## Особенности реализации

1. **Точная верстка** - дизайн сверстан 1 в 1 согласно макету Figma
2. **Адаптивность** - используется абсолютное позиционирование согласно макету
3. **Типизация** - все компоненты типизированы с помощью TypeScript
4. **Модульность** - каждый компонент изолирован и может быть легко изменен
5. **Готовность к расширению** - модальные окна готовы к реализации функционала

## Шрифты

Проект использует следующие шрифты из Google Fonts:
- **Bricolage Grotesque** (400, 500, 600, 800) - для заголовков
- **Inter** (300, 400, 500, 600) - для основного текста

Шрифты подключены через `index.html` и доступны глобально.
